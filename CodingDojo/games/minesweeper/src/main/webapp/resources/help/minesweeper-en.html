Developers defeat the virus!

A challenge for the mind and a bit of fun - it's about the updated version of the minesweeper game by the Lviv JS Community! We’re launching a coding game where instead of seeking bombs, you need to look for infected cells and send them to "quarantine." Write your rescue bot and fight with other virus fighters!

Who can participate?

EPAMers who are familiar with JavaScript, Java, C #, or Python and strive not only to have fun but also to improve approaches to algorithmic programming.

How to play?

Each player has his playing field and can move around in free cells in all four directions. The only way to know whether there are viruses around the cell is to get into it. The revealed number shows the number of infected cells around the current one. If there is no number, then everything is OK. If the player comes across an infected cell, the game is over. Therefore, you should consider in advance how to "clear" the area in a given direction.

To start, you need to download the client of your programming language and run it.

How to win?

For correctly moved quarantine cells the player receives bonus points, for an error - penalty points. After a mistake, the player appears on a new field with random placement of infected cells and plays first (because you need to save the planet!) - and so an infinite number of times. After adding all points, the player with the highest total points wins (as of a certain point in time).

Well, are you ready to fight viruses? :)

The game updates every second the server sends your client (bot) the status of the updated field at the moment and awaits a response - a command to the player. In the next second, the player must timely give the player a command. If you don’t do that on time, the player stands still.

There are several teams: UP, DOWN, LEFT, RIGHT - set the player in motion in a given direction by 1 cell; ACT + UP | DOWN | Left | RIGHT - set the flag in the given path.
